<a id="1-резюме"></a>
## 1. Резюме проекта

**Что строим.** Одностраничный лендинг для семейного ресторана MOMO в Севастополе. Цель — превратить случайного посетителя сайта в гостя ресторана за один сеанс: показать атмосферу, дать понять кухню и сразу предложить забронировать стол по телефону.

**Тип проекта.** Landing page, mobile-first, без личных кабинетов и онлайн-оплаты на MVP. Содержит динамический контент (Севастопольские сезоны), требует CMS-подхода для частых правок.

**Ключевые отличия от конкурентов.**
- Прямой вид на Севастопольскую бухту, передан на главном экране (фото `МОМО/интерьер_апрель2026/IMG_8853.HEIC`).
- Сегментирующий блок под Hero: «Сегодня я: с подругами / на свидание / с семьёй / на сезонный сет» — каждый сегмент видит свой оффер.
- Реальный фирстиль интерьера (коралл + мята + навы + пыльно-розовый + светлое дерево) — оригинальная палитра, не Instagram-пастель.
- Бренд-визуал лотосов как декоративный ключ (повторяет панно из реального интерьера).
- Звонок одним тапом + резервная форма «Перезвоните мне» вместо корпоративного «свяжемся в течение дня».

**Что НЕ строим.** Доставку, корзину, личный кабинет, тёмную тему, Telegram-бот бронирования (всё — V2/V3 или вне scope).

## 2. Цели сайта и KPI

### 2.1. Бизнес-цели
1. **Сформировать первое впечатление за 3 секунды** — передать атмосферу: свет, море, спокойствие, забота о госте.
2. **Закрыть бронь по телефону одним тапом** — главное конверсионное действие.
3. **Показать меню** — два основных раздела (Кухня и Бар) + Детское меню + Севастопольские сезоны.
4. **Снять трение для туристов** — адрес на карте, как добраться, режим работы, контакт.
5. **Перевести часть посетителей в долгий канал** — Telegram-канал и email-подписка для возвратов.

### 2.2. Целевые метрики (первые 6 месяцев после запуска)
| Метрика | Цель |
|---|---|
| Трафик / месяц | 5 000+ |
| Bounce rate | < 50% |
| Scroll-depth до меню | ≥ 40% |
| Клики по `tel:` CTA | ≥ 8% от посетителей |
| Заявки через форму «Перезвоните мне» | ≥ 2% |
| % дозвонов администратора (от tel-кликов) | ≥ 90% |
| Form-completion «Перезвоните мне» | ≥ 80% |
| Подписки на TG-канал | 200+ за 6 мес |
| Подписки на email | 100+ за 6 мес |
| Возвраты на сайт | ≥ 25% returning visitors |
| Lighthouse Performance / Accessibility / SEO | ≥ 90 / ≥ 95 / ≥ 95 |

<a id="3-аудитория"></a>
## 3. Целевая аудитория

Аудитория делится на **три первичных сегмента** местных жителей + один вторичный (туристы). Лендинг сегментирующий — отдельный оффер и CTA для каждого. Туристы — «мастхэв-локация» города, но **не главный сегмент**: основная выручка с местных.

| Приоритет | Сегмент | Что они хотят от MOMO | CTA сегмента |
|---|---|---|---|
| **Первичный 1** | **Женщины 45–60** (бухгалтер, педагог, риелтор) | Знакомое место, где комфортно и без напряжения. Тёплая атмосфера, спокойный обед или ужин с подругами или коллегами | «Встретиться с подругами» |
| **Первичный 2** | **Молодые пары 20–27** (интеллигентные, военные, моряки) | Безопасное место без лишнего внимания, среди комфортной публики. Романтика, не отвлекаться друг от друга | «Романтический вечер» |
| **Первичный 3** | **Молодые родители 25–35** (малый бизнес, военные, менеджеры) | Выход из дома и быта. Друзья с детьми. Семейные торжества и мини-банкеты | «Семейный обед / банкет» |
| Вторичный | **Турист с семьёй** (Москва, Питер, провинция, 30–50) | Красиво, с видом на море, проверенное место (4.9 на Яндексе) | «Стол у окна с видом» |

**Кого аудитория НЕ включает.** Молодёжь до 20 «на тусовку», большие шумные компании, корпоративы с танцами. Тон и эстетика сайта это не считают.

**Дополнительный канал — банкеты и торжества.** MOMO активно используется для семейных мероприятий, от свадеб до поминок. Это серьёзная статья выручки. На главной не выносится отдельной секцией (чтобы не разбавлять основной оффер), но **должна быть отдельная страница / якорь «Банкеты»** — со ссылкой из футера и из CTA-карточки «Семейный обед / банкет».

### 5.0. Шапка (sticky)
- Логотип MOMO слева (sans-serif wordmark с горизонтальными линиями).

- Справа: текст-ссылка **«+7 978 855-75-54»** + кнопка-CTA **«Забронировать»** (coral).
- На mobile: бургер-меню для текста, кнопка «📞» остаётся видимой.

**Визуал:** фото `МОМО/интерьер_апрель2026/IMG_8853.HEIC` на 100vw × ~90vh (desktop) / 100vw × ~80vh (mobile). Лёгкий тёмный градиент снизу для читаемости текста.

> Семейный ресторан с видом на бухту.

**CTA:**
- Основная кнопка (coral, заливка): **«Забронировать · +7 978 855-75-54»** → `tel:+79788557554`.
- Параллельная (navy, outline): **«Перезвоните мне»** → открывает inline-форму ниже на странице.

Под Hero — 4 карточки в одну строку (desktop) / 2×2 (tablet) / 1 колонка (mobile). Каждая ведёт к якорному скроллу + подсвечивает релевантный оффер. Заголовки в первом лице — «сегодня я…» — это разговор с гостем, не сегментация «вы — турист».

| Карточка | Иконка | Заголовок | Сабтекст | Ведёт к секции |
|---|---|---|---|---|
| 1 | две чашки кофе | **Встречаюсь с подругами** | Светлый зал, тихий джаз, спокойный разговор | → «О нас» + «Меню — Кухня» |
| 2 | бокал | **На свидание** | Стол у окна, мягкий свет, без лишнего внимания | → «О нас» + «Меню — Бар» |
| 3 | детская ладошка | **С детьми или с семьёй** | Детская комната, детское меню, стульчики, мультики | → «Детское меню» + «Для семей» |
| 4 | веточка | **На сезонный сет** | Локальные продукты, 4 блюда + 4 коктейля | → «Севастопольские сезоны» |

**Отдельная карточка-полоска под четырьмя:** «🎉 **Банкеты и торжества** — свадьбы, юбилеи, семейные встречи. Скажите при звонке» → smooth-scroll к якорному блоку «Банкеты» на главной (см. §5.9.1). **Отдельной страницы `/banket` НЕ делаем** — решение заказчика 2026-05-20.

**Правило 60/20/10/5/5:**
- 60% — off-white фон.
- 20% — navy / charcoal (текст, тёмные плашки).
- 10% — coral (CTA, ключевые акценты).
- 5–7% — teal (иконки, тэги).
- 3% — pink + oak (декор иллюстраций).

**Жёсткое правило:** Teal и Pink **никогда** не используются как текст на off-white (контраст 2:1 — FAIL по WCAG). Только заливки плашек, текст на них — navy или charcoal.

### 6.2. Типографика

**Display H1 (Hero + заглавия секций):**
- Шрифт: `PP Editorial New Light Italic` (или `Cormorant Garamond` как fallback)
- Размер: clamp(56px, 8vw, 120px)
- Цвет: на Hero — `--surface` (#FFFFFF) с тенью; внутри секций — `--navy`

**H2 (подзаголовки секций):** `PP Editorial New` regular, 40–48px, navy

**H3:** `Inter` semibold 24–28px, navy

**Body:** `Inter` regular 17–18px (mobile **минимум 17px**), line-height 1.6, navy

**Small / captions:** `Inter` 14px, ink-soft

**Logo wordmark MOMO:** sans-serif с горизонтальными линиями над и под буквами (например `Futura` / `Avenir Next Demi`). Воссоздаётся по образцу из реальных фото (`Детское меню/WVTzF_23U7...`).

**Ингредиенты в карточках меню:** `Inter Light` 12–13px, **uppercase**, tracking +60 — повторяет манеру оригинальных фирменных макетов.

**Опционально (V2):** рукописный акцент-шрифт (например `Catchy Mager`) для подписей под фото в стиле open-air кафе.

### 6.3. Декор и иллюстрации

- **Лотосы на navy** — ключевой бренд-визуал. SVG-иллюстрации в углах секций «О нас» и «Севастопольские сезоны». Лёгкая sway-анимация (1–2° rotate, 4с loop, респект `prefers-reduced-motion`).
- **SVG-noise overlay** `opacity: 0.04` на фон всего сайта — лёгкая «бумажная» текстура.
- **Иконки** — outline 1.5px, скруглённые концы. Никаких 3D-рендеров, никакого glassmorphism.
- **Скругления карточек:** 16–24px.
- **Скругления кнопок:** 999px (pill) — поддерживает «семейный» тон без острых углов.

### 6.4. Сетка и ритм

- Контейнер: max-width 1200px.
- Отступы: 24 / 48 / 96 px.
- Сетка: 12 колонок (desktop) / 6 (tablet) / 4 (mobile).
- Breakpoints: 360 / 768 / 1024 / 1440 px.
- Карточки меню: 1 колонка (mobile) / 2 (tablet) / 3 (desktop).

**Светлая дневная атмосфера** для основной части сайта: коралл, мята, светлое дерево, окна с морем. **Тёплый вечерний регистр** для секции «Согревающие напитки» в баре (`--velvet`).

- Alt-тексты у всех изображений (по содержанию кадра).
- `aria-label` для кнопок-иконок без текста.

## 12. Свод рекомендаций трёх экспертов

(Полная оценка — research.md «Экспертная оценка». Здесь — операционный свод того, что обязательно учесть.)

### 12.1. От UX-дизайнера

- Не один CTA — звонок + форма как параллельные пути.
- Меню — карточки на странице, не PDF.
- Никаких каруселей.
- Body-шрифт **минимум 17 px** на mobile.
- Pink и Teal **никогда** не используются как текст.
- Один sticky-элемент в зоне большого пальца — не плодить.
- Hero обязан содержать обещание (фото без слогана не работает).
- Часы и адрес в hero, не в подвале.
- Цены на каждой карточке.
- Аллергены — обязательно.
- FAQ перед финальным CTA.
- Реальные фото зала — у нас есть, использовать.

### 12.2. От маркетолога-CRO

- Не «универсальный оффер», а **4 сегмента** через карточки «Я зашёл как…».
- 7 триггеров доверия в Hero: фото, рейтинг 4.9 (не 5.0), часы, адрес, чек, телефон, годы работы.
- 8-шаговая воронка с метриками на каждом шаге.
- Точная микрокопия под CTA: «Звонок бесплатный. Ответим за 1–2 гудка».
- Sticky-CTA + 5 повторов.
- Лид-магнит для email — PDF «Гастрономический Севастополь от MOMO».
- Сертификаты с первого дня — отдельная воронка.

### 12.3. От потенциального клиента (владелец бизнеса 35–55)

- За 5 секунд видит 5 атрибутов: фото локации / цена / рейтинг / часы / меню. 4 из 5 — остался.
- Профессиональные фото обязательны (любительские = недоверие).
- Бронь в один тап без регистрации (звонок — да).
- «Подходит для встречи с партнёром» в тоне сайта (без слова «бизнес»).
- Карта вин с конкретными позициями и ценами.
- Английская версия — не V2, а MVP/fast-follow.
- Никакого пафоса «лучший ресторан Севастополя».
- Узнаваемая Tilda-эстетика 2018 = снижение доверия. Лучше Next.js, либо очень кастомизированная Tilda.

# ЧАСТЬ II. Telegram Mini App — пошаговый план разработки

Дата: 2026-06-12.
Источник: [research.md «Telegram Mini App — исследование»](research.md), §1–§7 (примеры, экраны, API) + «Экспертная оценка TMA» (UX, маркетолог, TMA-разработчик).

> Это **продолжение** brief.md, не замена. ЧАСТЬ I — это лендинг. ЧАСТЬ II — это Mini App. Они работают параллельно: лендинг для туристов и поискового трафика, TMA — для местных постоянных гостей через Telegram. Контент и фото переиспользуются.

## TMA-1. Что строим (scope MVP)

**Цель.** Telegram Mini App, в котором гость МОМО:
1. Видит атмосферу, меню и сезонный сет за 5 секунд.
2. Бронирует стол за 1 тап через `requestContact` (вместо формы с маской телефона).
3. Может позвонить администратору одним тапом.
4. Может посмотреть, как добраться, через нативные карты.

**Внешний URL.** TMA = одностраничное приложение, доступное:
- Из бота МОМО (`@momo_sev_bot`) — кнопка меню Telegram «Открыть МОМО»
- По прямой ссылке `https://t.me/momo_sev_bot/app` (можно отправлять в чаты)
- Через QR-код в зале ресторана (deep link с параметром, например `?startapp=table_12`)

**Должно выглядеть как приложение, не как сайт.** Это значит:
- Никакого sticky-header с логотипом — у Telegram уже есть шапка
- Никакого cookie-баннера — в webview не нужно
- Никакой кнопки «Наверх» — есть BackButton от Telegram
- Главное действие — **MainButton Telegram (нативная кнопка снизу)**, а не свой `<button>` в макете
- Тема (светлая/тёмная) подхватывается из Telegram автоматически
- Тактильный отклик `HapticFeedback` на каждый тап по кнопке
- Открытие внешних ссылок — через `WebApp.openLink()`, не `target="_blank"`

// На старте app
WebApp.ready();

## TMA-5. Дизайн-система TMA (Telegram-native)

### Цвета — двойная система

**1. Тема Telegram (динамически меняется):**
- `--tg-bg-color` = `themeParams.bg_color` (фон секций)
- `--tg-text-color` = `themeParams.text_color` (основной текст)
- `--tg-hint-color` = `themeParams.hint_color` (мелкий серый текст)
- `--tg-button-color` = `themeParams.button_color` (для elements adapt)
- `--tg-button-text-color` = `themeParams.button_text_color`

**2. Бренд-цвета МОМО (фиксированные):**
- `--coral: #E07856` — основные акценты, кнопки CTA
- `--teal: #7FB8B0` — вторичные акценты, иконки
- `--navy: #1F3045` — header color через `WebApp.setHeaderColor`
- `--pink: #E8B6B0` — мягкие плашки (банкеты)
- `--teal-kids: #5BB8B5` — детское меню
- `--velvet: #5C3A4A` — секция согревающих напитков

**Правило миксования:** базовые фон/текст подхватываются из Telegram (light/dark), бренд-цвета МОМО — только для CTA, акцентных плашек, иконок. **Текст с цветами бренда всегда явно прописан** (не наследует от темы).

### Типографика — system-fonts

```css
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif;
```

| Элемент | Размер |
|---|---|
| Заголовок экрана (в шапке Telegram) | автоматически |
| H1 (Hero, экраны) | 28–32 px, weight 600 |
| H2 (секции) | 20–22 px, weight 600 |
| Body | **17 px** (минимум), line-height 1.5 |

| Микрокопия | 12 px, серый |

### Кнопки

- **MainButton (Telegram native, не html)** — основной CTA на каждом экране. Текст ≤ 32 символов. Цвет coral.

## TMA-6. Чего НЕ будет в первой версии (MVP)

| Функция | Где будет | Почему не сейчас |

| Онлайн-заказ еды / доставка | НИКОГДА | У МОМО нет доставки, не сценарий |

| Карта лояльности «штампы» (CloudStorage) | V2 | После запуска MVP, как только будет 100+ постоянных пользователей |
| Push о смене сезонного меню (через бота) | V2 | Требует `requestWriteAccess` + контент-стратегию TG-канала |
| `shareToStory` для блюд | V2 | Требует медиа на CDN, не критично для MVP |
| `addToHomeScreen` (иконка МОМО на главном) | V2 | На MVP не критично, нужно после удержания базы |
| Календарь брони с выбором даты/времени/гостей | V2 | На MVP — `requestContact` + перезвон. Календарь = другая воронка |
| FAQ-секция | V2 (если будут вопросы) | В TMA антипаттерн — длинные тексты |
| Поиск по меню | V2 | Когда позиций станет > 60 |
| Фильтры меню (веган / без глютена / детское) | V2 | Не первичная потребность |
| Yandex.Метрика / GA | НЕТ | В webview глючит. Используем `@telegram-apps/analytics` |
| Регистрация / аккаунт пользователя | НИКОГДА | Есть `initData`, всё |
| Биометрия | V3 | Только если будут крупные платежи |
| Геолокация «вы рядом» | V3 | Маркетинговая фишка, не критична для MVP |
| Английская версия | V2 | Сначала русская, потом локализация |
| Чат с администратором внутри TMA | НЕ ДЕЛАЕМ | Есть `openTelegramLink` на админ-чат |
| 3D-тур по залу | V3 | После расширения галереи |
| Свой login/OAuth | НИКОГДА | Telegram даёт авторизацию через initData |
| Cookie-баннер | НИКОГДА | В TMA не нужен |
| Тёмная тема как переключатель | НЕТ | Подхватываем из Telegram |
| Sticky-header с логотипом | НЕТ | Дублирует Telegram-шапку |

## TMA-9. Сроки и стоимость (оценка)

### Этап TMA-1. Подготовка инфраструктуры (3–5 дней)
- Создание бота через BotFather
- Настройка Cloudflare Worker для бэкенда бота
- Подготовка хостинга на Vercel
- Стартер `npm create @telegram-apps/mini-app@latest`

### Этап TMA-2. Разработка MVP-экранов (10–14 дней)
- 9 экранов + 2 bottom sheet
- Интеграция Telegram SDK (BackButton, MainButton, HapticFeedback, themeChanged)
- `requestContact` + бот-flow для брони
- Подключение `@telegram-apps/analytics`

### Этап TMA-3. Контент + оптимизация (3–5 дней)
- Перенос фото из лендинга (МОМО/, Севастопольские сезоны/)
- Конвертация в WebP, оптимизация ≤ 150 КБ ключевые
- Self-host системных шрифтов (или fallback к -apple-system)
- Статичная картинка карты Яндекса

### Этап TMA-4. QA и запуск (2–3 дня)
- Тестирование по чек-листу §8
- Smoke-test на 5 устройствах
- Передача доступов заказчику

### Итого
- **18–27 рабочих дней** (~4–5 недель календарных)
- **Бюджет:** 200–350 К ₽ единоразово (зависит от подрядчика) + 0 ₽/мес инфраструктура (бесплатные планы Vercel + Cloudflare Workers)

### V2 (через 1–3 месяца после запуска)
- CloudStorage-лояльность («штампы»)
- Push о смене сезонного меню
- shareToStory на карточках блюд
- Календарь брони
- addToHomeScreen

### V3 (длинный горизонт)
- Bot Payments через ЮKassa (сертификаты, депозиты банкетов)
- Геолокация, биометрия, геймификация
